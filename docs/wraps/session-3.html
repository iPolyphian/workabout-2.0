⚖️ Decisions
URL searchParams as single source of truth for filters
Problem Needed a state management approach for search filters that works with SSR and is shareable/bookmarkable
Solution Custom useSearchParamsState hook wraps Next.js useSearchParams + useRouter. No external state library. Prices in URL are GBP, internal state uses pence.
Negative margins for edge-to-edge map
Problem AppShell adds px-6 py-6 padding but the map needs to fill to the right edge
Solution Search layout uses -mx-6 -my-6 to override, avoiding changes to the shared AppShell component.
💡 Learnings
Server/client boundary for pure functions
Discovery parseSearchFilters lived in a "use client" hook file but was called from a server component, causing a runtime error
Change Extracted pure functions to src/lib/search-params.ts (no directive). Re-exported from the hook file for backward compat. Rule: pure utils must not live in "use client" files if called from server components.
